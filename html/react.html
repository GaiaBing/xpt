重点

1.组件和props

组件使你可以将UI划分为一个一个独立、可复用的小部件，并可以对每个部件进行单独的设计。组件名称总是以大写字母开始。

1). 分类：函数式组件和类组件

函数式组件

function Welcome (props){
    return <h1>Hello, {props.name}</h1>
}

类组件

class Welcome extends React.Component{
    render(){
        return <h1>Hello, {props.name}</h1>
    }
}

比较：函数式组件更简洁；类定义的组件使用ES6写法，有局部状态

2). ReactDOM.render() 渲染组件

参数1：组件，参数2：父节点对象

3). props不可修改

所有React组件都必须是纯函数，并禁止修改自身props

注意：render中只能有一个根标签

2.state和生命周期

1). ES5和ES6在state写法上的区别参考上面网址中的写法对照表

在constructor()中初始state

用setState()改变state

setState可以接收一个函数，这个函数接受两个参数，第一个参数表示上一个状态值，第二个参数表示当前props

this.setState((prevState, props) => ({
  counter: prevState.counter + props.increment
}));

注意：直接修改state不会重新渲染

2). React 组件生命周期

用于在渲染dom前后或者移除dom前后插入事件

有三个状态：Mounting（已插入真实 DOM）、Updating（正在被重新渲染）、Unmounting（已移出真实 DOM）

注意：React事件使用驼峰命名，而不是全部小写
